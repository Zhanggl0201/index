# YunfanTech 主页站 · EdgeOne Pages 部署与访问防护指南

本项目已接入 **EdgeOne Pages 边缘中间件**，实现**真正的服务端访问防护**：
未登录的访客在 CDN 边缘节点就会被拦截，**连主页 HTML、图片都拿不到**——
这与之前纯前端的 sessionStorage 登录门有本质区别（后者代码全在浏览器，随手就能绕过）。

---

## 一、防护原理

```
访客请求任意页面
        │
        ▼
┌─────────────────────────────┐
│  middleware.js（边缘节点执行）  │
│  1. 白名单？(登录页/登录接口)   │──是──▶ 放行
│  2. Cookie 里有合法凭证？       │──是──▶ 放行（返回真实页面）
│  3. 都不满足                    │──否──▶ 重定向到 /login.html
└─────────────────────────────┘
```

- 账号密码在**边缘节点服务端**校验，前端源码里看不到密码。
- 登录凭证是 **HMAC-SHA256 签名 token**，无法伪造。
- 凭证写入 **HttpOnly + Secure + SameSite Cookie**，JavaScript 无法读取/窃取，有效期 8 小时。

## 二、文件结构

```
blog-homepage/
├── middleware.js            ← 边缘鉴权中间件（拦截整站）
├── functions/
│   └── api/
│       ├── login.js         ← POST /api/login   校验账密、签发凭证
│       └── logout.js        ← POST /api/logout  清除凭证
├── login.html               ← 玻璃拟态登录页
├── index.html               ← 主页（已移除旧的前端登录门）
└── assets/images/           ← logo、轮播图
```

## 三、部署步骤

### 1. 上传代码
把整个 `blog-homepage` 目录推到你的 Git 仓库（GitHub / GitLab / Gitee 均可），
或直接用 EdgeOne Pages 控制台「上传文件夹」。

### 2. 在 EdgeOne Pages 新建项目
- 控制台 → Pages → 创建项目 → 关联仓库（或上传目录）。
- **构建命令**：留空（纯静态 + 边缘函数，无需构建）。
- **输出目录**：项目根目录（即 `blog-homepage`，里面有 `index.html`）。
- EdgeOne 会自动识别根目录的 `middleware.js` 和 `functions/` 目录。

### 3. ⭐ 配置环境变量（关键安全步骤）
控制台 → 项目 → 设置 → **环境变量**，添加以下三项：

| 变量名        | 说明                          | 示例值                              |
|--------------|-------------------------------|-------------------------------------|
| `AUTH_USER`  | 登录用户名                     | `admin`                             |
| `AUTH_PASS`  | 登录密码                       | `你的强密码`                         |
| `AUTH_SECRET`| 凭证签名密钥（**务必改**）      | 一长串随机字符，如 `k9Xz...48位随机` |

> **必须配置，否则站点不可用（fail-closed）**：代码里**不再保留任何默认账号 / 密码 / 密钥**。
> 若这三个环境变量未配齐：中间件会对整站返回「站点未配置访问凭证」提示页，登录接口也会返回 500。
> 这样即使源码公开（本项目在公开仓库），也没有任何可用的默认凭证被人利用。
> 真实账密和密钥只存在于 EdgeOne 控制台。
>
> `AUTH_SECRET` 生成一串随机字符即可，例如在终端跑：
> `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`

### 4. 部署 & 绑定域名
- 点击部署，等待完成。
- 建议绑定你自己的域名（控制台 → 域名管理），并开启 HTTPS。
  （凭证 Cookie 带 `Secure`，必须走 HTTPS 才会生效。）

## 四、验证防护是否生效

1. **无痕窗口**打开你的站点 → 应自动跳到 `/login.html`。
2. 直接访问 `你的域名/index.html` → 同样被拦截跳登录页（前端门做不到这点）。
3. 直接访问某张图片 URL（未登录）→ 返回 `401 Unauthorized`。
4. 输入正确账密 → 跳回主页，正常浏览。
5. 点右上角「退出」→ 凭证清除，再次访问又需登录。

## 五、常见问题

**Q：改了账号密码要重新部署吗？**
A：不用。改环境变量后，EdgeOne 会让新配置在下次请求生效（部分情况需触发一次重新部署）。

**Q：登录后一会儿又要重新登录？**
A：凭证有效期默认 8 小时，可在 `functions/api/login.js` 和 `middleware.js` 里调整 `MAX_AGE`。

**Q：本地怎么调试？**
A：安装脚手架 `npm i -g edgeone`，项目根目录执行 `edgeone pages dev`，
   即可在本地模拟边缘函数（含中间件）。

**Q：还想更强的防护？**
A：可叠加 EdgeOne 的 WAF、IP 黑白名单、地域封禁等安全能力；
   也可把账密校验换成对接你自己的用户系统 / OAuth。

## 六、安全须知

- 部署上线前，**务必**在控制台配齐 `AUTH_USER` / `AUTH_PASS` / `AUTH_SECRET`，否则站点会 fail-closed（整站显示未配置提示、无法登录）。代码内已无任何默认账密。
- 不要把真实密码提交进 Git 仓库——用环境变量。
- 全程使用 HTTPS，否则 `Secure` Cookie 不生效。
